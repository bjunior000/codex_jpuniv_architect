# GitHub 업로드 가이드

이 폴더는 GitHub 저장소에 그대로 올려도 읽을 수 있게 정리되어 있다.

## 1. GitHub에서 저장소 만들기

- GitHub 로그인
- 우측 상단 `+` 클릭
- `New repository` 선택
- 저장소 이름 입력
  - 예: `japan-architecture-admissions`
- `Public` 선택
- `Create repository` 클릭

## 2. 웹에서 바로 업로드하는 방법

가장 쉬운 방법:

- 새 저장소 화면에서 `uploading an existing file` 클릭
- 현재 폴더의 파일과 `tier1`, `tier2` 폴더를 끌어다 놓기
- 커밋 메시지 입력
  - 예: `Add Japanese university architecture admissions notes`
- `Commit changes` 클릭

## 3. 업로드 후 보는 방법

- 저장소 메인에서 `README.md`가 첫 화면에 보인다.
- `tier1`, `tier2` 폴더를 눌러 문서를 탐색하면 된다.
- Markdown 파일은 GitHub가 자동으로 읽기 좋은 형태로 보여 준다.

## 4. 가족이나 동생에게 공유하는 방법

- 저장소 주소를 그대로 보내면 된다.
- 예시 형식:
  - `https://github.com/사용자이름/저장소이름`

## 5. 추천 사항

- 저장소 설명(Description)에 `일본 대학 건축계열 외국인 유학생 모집요강 정리` 정도를 적어두면 좋다.
- 저장소 첫 화면은 루트 `README.md`가 담당하므로, 계속 이 파일을 목차처럼 유지하면 된다.
- 새 학교를 추가할 때는 `tier1` 또는 `tier2`에 파일을 넣고 README에 링크를 추가하면 된다.
